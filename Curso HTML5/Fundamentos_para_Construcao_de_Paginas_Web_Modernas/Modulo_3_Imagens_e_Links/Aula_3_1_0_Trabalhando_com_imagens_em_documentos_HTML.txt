3.1 Trabalhando com imagens em documentos HTML
Incorporar imagens em um arquivo HTML é uma prática fundamental para enriquecer o conteúdo visual de uma página web, tornando-a mais atraente e informativa para os usuários. Para garantir que as imagens em seus arquivos HTML sejam carregadas corretamente e exibidas sem problemas, é essencial seguir algumas recomendações gerais. Essas recomendações ajudam a assegurar que as imagens sejam acessíveis e funcionem conforme o esperado em diferentes navegadores e dispositivos (MDN Web Docs, 2024).

Neste contexto, ao longo deste documento, recomendamos que você nomeie suas pastas e arquivos utilizando apenas letras minúsculas e sem espaços. Muitos servidores web são case sensitive, o que significa que diferenciam letras maiúsculas de minúsculas. Por exemplo, se você armazenar uma imagem como MyImage.jpg e tentar acessá-la como myimage.jpg, o link não funcionará. Portanto, é crucial manter consistência nos nomes dos arquivos.

Além disso, esses servidores também podem enfrentar dificuldades ao lidar com espaços nos nomes dos arquivos. Muitos servidores substituem espaços por "%20" (o código para espaço em URLs), o que pode quebrar links e causar erros. Para garantir compatibilidade com todos os sistemas, é recomendável substituir espaços por traços (-). Nesse contexto, sugerimos nomear suas imagens como "minha-imagem.jpg" em vez de "Minha Imagem.jpg" e suas pastas como "site-teste/imagens" em vez de "site-teste/Imagens do Site".

Para evitar problemas futuros e assegurar maior compatibilidade e eficiência, acostume-se a nomear suas pastas e arquivos em letras minúsculas, sem espaços, e utilizando traços para separar palavras. Essa prática não apenas melhora a consistência, mas também aumenta a confiabilidade do seu site.
